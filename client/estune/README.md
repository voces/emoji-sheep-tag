# estune (vendored)

This directory is a self-contained drop-in copy of the estune adaptive
music engine. Generated by `scripts/build-vendor.ts` in the estune
source repo.

## What's here

- TypeScript source for the v3 engine (~25 files).
- `segmentSources.generated.ts` — bed, hook, and stinger texts inlined as
  string constants. Replaces Vite `?raw` imports so this works in any
  TS toolchain.
- `index.ts` — re-exports the public API.

## What's NOT here

- The SF2 soundfont (`GeneralUser-GS.sf2`, ~5MB). Vendor it as a
  separate asset; the engine loads it on first audio activation. In
  the source repo it lives at `public/GeneralUser-GS.sf2`.

## API surface

Three driver calls plus engine lifecycle. The game produces facets directly
(no raw inputs); the engine smooths them temporally.

### Per-tick: `setGameState(engine, state)`

`state` shape:
```ts
interface FullGameState {
  state: "lobby" | "build" | "active";
  perspective: "sheep" | "wolf" | "spirit";
  sheep?: SheepFacets;        // when perspective is "sheep" or "spirit"
  wolf?: WolfFacets;          // when perspective is "wolf"
  roundHook: Hook | null;
  mode?: "survival" | "vip" | "switch" | "vamp" | "bulldog";
}

interface SheepFacets {
  alive: boolean;
  threat: number;        // 0-1; game computes from however many wolves it wants
  agency: number;        // -1..+1; team dominance
  isolation: number;     // 0-1; alone-and-vulnerable feel
  lastAlive: boolean;
  roundProgress: number; // 0-1
}

interface WolfFacets {
  proximity: number;     // 0-1; hunting intensity, game-computed
  agency: number;        // -1..+1; pack dominance
  isolation: number;     // 0-1; pack scatter (pass 0 if not tracked)
  roundProgress: number; // 0-1
}
```

The engine derives `failing` (desperate-bed switch) internally from
agency + roundProgress. The game doesn't compute it.

### Discrete events: `fireGameEvent(engine, event)`

Four event types. Structural transitions (lobby→build→active) are NOT
events — the engine sees them in `state.state` and fires bed-bridge
stingers automatically.

```ts
type GameEvent =
  | { type: "capture"; subject: "self" | "ally" }
  | { type: "rescue"; subject: "self" | "ally" }
  | { type: "assassination-attempt"; subject: "self" | "ally" }
  | { type: "round-end"; winner: "sheep" | "wolf" };
```

### Round setup: `setRoundHook(engine, hook, library)`

Once per round, after building the bed library:

```ts
import {
  createV3Engine, setGameState, fireGameEvent, setRoundHook,
  buildBedLibraryFromText,
  BED_SOURCES, HOOK_SOURCES,
} from "./estune/index.ts";

const engine = createV3Engine([]);
const lib = buildBedLibraryFromText(
  HOOK_SOURCES["r2s-ascending"],   // sheep-side default hook
  [
    { bed: "building",    text: BED_SOURCES.building,    rootName: "G", beatsPerBar: 4 },
    { bed: "cautious",    text: BED_SOURCES.cautious,    rootName: "G", beatsPerBar: 4 },
    { bed: "terror",      text: BED_SOURCES.terror,      rootName: "G", beatsPerBar: 4 },
    { bed: "hero",        text: BED_SOURCES.hero,        rootName: "G", beatsPerBar: 4 },
    { bed: "spirit",      text: BED_SOURCES.spirit,      rootName: "G", beatsPerBar: 4 },
    { bed: "patrolling",  text: BED_SOURCES.patrolling,  rootName: "E", beatsPerBar: 4 },
    { bed: "stalking",    text: BED_SOURCES.stalking,    rootName: "E", beatsPerBar: 4 },
    { bed: "attack",      text: BED_SOURCES.attack,      rootName: "E", beatsPerBar: 4 },
    { bed: "desperate",            text: BED_SOURCES.desperate,            rootName: "E", beatsPerBar: 4 },
    { bed: "desperate-frustrated", text: BED_SOURCES["desperate-frustrated"], rootName: "E", beatsPerBar: 4 },
    { bed: "lobby",                 text: BED_SOURCES.lobby,                       rootName: "G", beatsPerBar: 4 },
    { bed: "build-sheep-bulldog",   text: BED_SOURCES["build-sheep-bulldog"],      rootName: "G", beatsPerBar: 4 },

setRoundHook(engine, /* hook */ null as any, lib);
```

## Game-side facet computation

The game is responsible for computing facets — the engine deliberately
doesn't include distance heuristics. Suggested aggregations for
multi-entity scenarios:

- **threat / proximity from N wolves**: soft-min via reciprocal sum
  `1 - 1/(1 + Σ DANGER_RADIUS/d_i)`, or sum-of-Gaussians
  `clamp01(Σ exp(-d_i² / σ²))`. Smoother than nearest-distance under
  motion.
- **isolation**: distance to nearest ally / typical-pack radius.
- **lastAlive**: `alive && livingCount === 1`.
- **agency**: team dominance from whatever score model the game uses
  (kills - deaths, % captured, etc.), normalized to -1..+1.

## Updating

Re-run `scripts/build-vendor.ts` in the estune source repo, then copy
the regenerated `vendor/estune/` over the consumer's vendored copy.
The directory is fully owned by the build script — local edits will be
overwritten on next build, so file bugs against estune upstream rather
than patching here.
